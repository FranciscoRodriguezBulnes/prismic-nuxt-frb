# Prismic + Nuxt Minimal Starter

Want to quickly get started building your own project with Prismic and Nuxt?  
This project includes basic configurations and nothing else.  
The project includes one Rich Text Slice, a homepage, and a dynamic page.

Demo: Open live demo  
Learn more: Prismic Nuxt Documentation

---

## 🚀 Quick Start

To start a new project using this starter, run the following commands in your terminal:

npx degit prismicio-community/nuxt-starter-prismic-minimal your-project-name  
cd your-project-name  
npx @slicemachine/init@latest

These commands will:

- Start a new Nuxt project using this starter
- Ask you to log in to Prismic or create an account
- Create a new Prismic content repository with sample content

When you're ready to start your project:

npm run dev

## 🧩 How to use your project

### Create a page

In Prismic:

1. Click the green pencil icon
2. Select Page
3. Add and rearrange Slices

Your new page will be accessible by its URL.  
To show it in navigation, add it manually.

## 👁 Preview documents

If you created your repository from this starter, previews are already configured for:

http://localhost:3000

To add a production preview:

1. Go to Settings → Previews
2. Create a new preview
3. Use /api/preview as the route

## 🛠 Customize this website

This project uses:

- @nuxtjs/prismic
- @slicemachine/adapter-nuxt
- Slice Machine

Important files:

- nuxt.config.ts — Prismic configuration
- pages/index.vue — homepage
- pages/[uid].vue — dynamic pages
- slices/\*/index.vue — Slice components

Do not edit:

- pages/slice-simulator.vue
- Slice model files generated by Slice Machine

# 📌 Guía ultra‑minimalista Nuxt + Prismic

### 🟦 1) Si algo falla de repente → borrar .nuxt

rm -rf .nuxt  
npm run dev

### 🟧 2) Si Prismic se desactiva → revisar slicemachine.config.json

Debe contener:

"repositoryName": "tu-repo"

### 🟩 3) Si hay errores de dependencias → borrar node_modules

rm -rf node_modules package-lock.json  
npm install

### ⭐ Regla de oro

1. Borrar .nuxt → 2) Revisar slicemachine.config.json → 3) Borrar node_modules solo si es necesario
